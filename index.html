Git Flow Cheatsheet

Making a New Project (local)

In the Terminal:

  1. mkdir to create a new directory
  2. cd into the new directory
  3. git init to track files in the new directory
  4. Add your files, make initial changes, etc
  5. git add . to track files with git
  6. git commit -m "[your commit message]" to create your initial commit
  7. Make more changes, add the files, commit the changes, and repeat!

Making a New Project (remote)

  1. In the browser, go to github
  2. Find the green new button
  3. Give your repo a name and click
  4. Copy the SSH or HTTPS link provided in the repo
  5. If you've already created a local project, follow the steps to add this github repo as a reomte, and push up the local code
  6. If you have not created a local project already, you can git clone [git url] the new Github repo in your terminal, and begin adding files.

Making Local Changes

After writing code in your editor, go to the terminal:

  1. Run git status to see what files have been changed
  2. Run git add [files to add] to stage those files for a commit
  3. Commit the changes with git commit -m "[descriptive commit message]"

Pushing Up Local Changes

  1. In your local project directory, make sure you have a remote repository connected
    • Run git remote -v to see if you have any remotes listed
    • If so, go to step 2
    • If not, grab the github url from your remote repo (or follow the steps above to create a new remote repo on github)
    • In your terminal, run git remote add origin [github url]
  2. Add and commit all changes you want to push
  3. Push the changes up to github
    • If you've never pushed this branch up before: git push -u origin [current branch name]
  4. If you have pushed this branch up before: git push

Forking and Cloning Down a Repo

  1. On github, find the repo you want to fork
  2. At the top right of the page, click the button
  3. Copy the SSH or HTTPS github url from your newly forked repo
  4. In the terminal, run git clone [github url]to clone the fork from github on to your local machine
  5. Follow instructions above for making, committing and pushing local changes

Branching

It's a best practice to work on separate branches for different features in a project. Branches are like sandboxes for development, where you can safely make changes and try things without possibly breaking the "production-ready" pieces of code (usually the Master Branch. This is how to make and work with git branches:

  1. In your terminal, run git branch to see a list of all local branches
  2. To make a new branch, run git branch [new branch name]
  3. To switch to an existing branch, run git checkout [branch name]
  4. To create and switch to a new branch at the same time, run git checkout -b [new branch name]
  5. Make and commit changes to your branch
  6. When you're ready to merge the branch into your master, push the local changes up to github (see instructions above), and make a PR (see below)

Making a PR

Once you've pushed local changes up to github, either from a branch or a fork, you'll need to make a Pull Request (PR) to get those changes encorporated into the production-ready part of the repo (this is usually the master branch)

  1. Push up your changes to github
  2. Navigate to the repo you just pushed to
  3. You should see a text box prompting you to "Make a Pull Request".
    • If not, you can click the button
  4. Pick which branch you want to merge, and where you want to make the merge to
  5. Write a detailed description of your pull request, using your PR template
  6. Tag group members to request a review, and click
  7. Your group members will review the PR, then merge it to the correct branch when it's ready!
